Before / After

The difference is not storage. It is control.

Password managers store secrets. Dotenv files scatter them. Vibe Vault turns secret access into a deliberate, audited part of the AI-coding workflow.

Before Vibe Vault

  • -Production tokens copied from a password manager into terminals and chats.
  • -.env files spread across repos with no central view or rotation context.
  • -No record of whether Cursor, Claude Code, or a shell command read a secret.
  • -Provider sync is manual, fragile, and easy to forget after a key rotates.

After Vibe Vault

  • +One local vault becomes the source of truth for AI-coding credentials.

  • +Every read lands in an audit log with agent, project, and timestamp.
  • +Cloudflare, Vercel, and PushCI sync happen when you explicitly choose.

Three-second workflow

Scan the repo. Prepare the agent. Run with secrets. Audit the read.

The CLI and macOS app share one vault, so the path from missing secret to verified agent access stays short and legible.

~/luna-worker
  • $vibevault scan
  • $vibevault cursor prepare
  • $vibevault run -- npm run dev
  • $vibevault push --to cloudflare

The setup command wires MCP, agent rules, skills, and a git leak guard so the AI tool receives permissioned access instead of raw copy-paste.

FAQ

Short answers for teams before they install.

The product stays intentionally narrow: local vault, agent audit, workflow setup, provider sync, and offline licensing.

Does this replace ChatGPT Projects or Claude Projects?

No. Those tools organize context and instructions. Vibe Vault protects the credentials those workflows need when agents run real commands.

Where do secrets live?

Secrets are encrypted locally. The vault ciphertext lives on disk and the master key is held in macOS Keychain.

Does Team phone home?

No. Offline VV1 licenses verify against an embedded public key. The app does not need a license server to open.

What does vibevault cursor prepare do?

It wires the MCP server, installs the agent skill and rules, and prepares guardrails so Cursor can request secrets through the vault.

Can it sync provider environments?

Yes. Vibe Vault can push to Cloudflare, Vercel, and PushCI when you explicitly run a sync command.
