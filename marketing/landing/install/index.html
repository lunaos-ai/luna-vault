CLI-first launch path

Install Vibe Vault without waiting for the notarized Mac app.

The local CLI, MCP server, scanner, git guard, browser host, and core vault package can ship source-first. The native DMG is held until Developer ID notarization is available.

Build from source Security architecture

Recommended now

Build the CLI and MCP server from source.

This avoids asking security-minded users to bypass Gatekeeper for an unnotarized app. It also gives HN and developer communities an inspectable path.

01
Clone and build 
git clone https://github.com/lunaos-ai/luna-vault
cd luna-vault
swift build -c release --product vibevault
swift build -c release --product vibevault-mcp
02
Run the scanner first 
.build/release/vibevault scan
.build/release/vibevault scan --git-only
.build/release/vibevault guard install
03
Prepare Cursor or MCP clients 
.build/release/vibevault cursor prepare
.build/release/vibevault mcp install --client all
Native app status: the macOS menu-bar app exists, but the public DMG is not promoted until it is signed with Developer ID, submitted to Apple notarization, stapled, and accepted by Gatekeeper.

Homebrew

Use the tap once it is published.

The Homebrew formula is prepared in the repo. Use this path after the tap is reachable from a clean machine. 

brew tap finsavvyai/tap
brew install vibevault
vibevault --version

Why no DMG yet?

Apple Developer ID is required for the public Mac app path.

Apple requires Apple Developer Program membership to request Apple-issued signing certificates. Developer ID distribution also requires a Developer ID certificate and notarization before Gatekeeper-safe public distribution.
